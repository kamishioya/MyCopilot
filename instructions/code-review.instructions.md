# コードレビュー基準

## レビュー優先度順
1. **🔴 セキュリティ**: 脆弱性、秘密情報漏洩、認証認可の問題
2. **🔴 正確性**: ロジックエラー、データ破壊リスク、競合状態
3. **🟡 コード品質**: SOLID原則違反、過度な重複、複雑性
4. **🟡 テスト**: クリティカルパスのテスト不足
5. **🟡 パフォーマンス**: N+1クエリ、メモリリーク等のボトルネック
6. **🟢 可読性**: 命名、ネスト深度、マジックナンバー
7. **🟢 ドキュメント**: コメント不足、README未更新

## チェックポイント
- 関数は小さく焦点を絞っているか（20-30行以内推奨）
- 深いネストがないか（最大3-4レベル）
- エラーハンドリングが適切か（サイレント失敗は禁止）
- 入力バリデーションが早期に行われているか
- パラメータ化クエリが使用されているか
- テストが Arrange-Act-Assert パターンに従っているか
- `skills/commenting/SKILL.md` の基準に従い、複雑なロジックへ理由コメントが付いているか
- 公開APIコメントが責務・前提条件・副作用を正しく反映しているか
- TODO / FIXME に担当者、日付、意図が含まれているか
