# 仕様書作成スキル

## 目的
`${input:SpecPurpose}` に関する仕様書を作成します。

AI（生成AI）が正確に理解・実行できるよう、明確で構造化された仕様書を生成します。

## AI最適化された仕様書の原則

- 正確で明示的、曖昧さのない言語を使用する
- 要件・制約・推奨事項を明確に区別する
- 見出し、リスト、テーブルなど構造化フォーマットを活用する
- すべての略語・専門用語を定義する
- 具体例とエッジケースを含める
- 外部コンテキストに依存しない自己完結型にする

## 保存先

仕様書は `.github/docs/spec/` ディレクトリに保存し、以下の命名規則に従うこと:
`spec-[目的]-[名前].md`

目的プレフィックス: `schema`, `tool`, `data`, `infrastructure`, `process`, `architecture`, `design`

## テンプレート

```md

title: [仕様のタイトル]

last_updated: [YYYY-MM-DD]
owner: [担当者/チーム]
```
