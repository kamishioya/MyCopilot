# HOW

## 言語別の基本形式

### C#

- 公開API: XML ドキュメントコメント
- ロジック説明: `// WHY:` 形式の行コメント

### TypeScript

- 公開API: JSDoc
- ロジック説明: `// WHY:` 形式の行コメント

### Python

- 公開API: Docstring
- ロジック説明: `# WHY:` 形式の行コメント

## レビュー手順

1. 差分のうち複雑なロジックを先に拾う
2. 該当箇所に理由コメントがあるかを見る
3. コメント内容と実装が一致しているか確認する
4. コメントが What の言い換えだけなら修正を要求する
5. TODO / FIXME に担当者と日付があるか確認する
